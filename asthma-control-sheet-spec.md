# 重症喘息 コントロール整理シート — 仕様書 (SPEC)

> Claude Code 実装用の仕様書。既存の参考実装 `asthma-control-sheet.html` を一緒に渡すと再現が速い。

---

## 1. 概要・目的

- **何のツールか**：重症喘息患者の重症度・コントロール状況を、1画面でタップ入力して整理するシート。
- **使う人/場面**：MRが医師との面談中に操作し、画面を一緒に見ながら、その患者の状態を埋めていく。
- **狙い**：コントロール不良・好酸球性の重症喘息で、かつ通院・治療継続の負担が大きい患者（＝26週毎投与の生物学的製剤が適する患者像）のニーズを可視化する。
- **配布モデル**：「**配って使う型**」。各自がHTMLのコピーを持ち、自分用に編集して使う。共有はHTMLファイルの**書き出し→配布**で行う。サーバー・共同編集はしない。

---

## 2. 技術制約（最重要）

- **単一の自己完結HTMLファイル**で完成させる。HTML / CSS / JS を1ファイルに内包する。
- 外部ライブラリ・CDN・フレームワーク・**ビルド工程・サーバーすべて不要**。ダブルタップ（ファイルを直接開く）で動作し、**オフラインで完結**する。
- ※開発時に何らかのツールを使う場合でも、**最終成果物は CSS/JS をインライン化した 1 つの `.html`** にすること。これを崩すと配布モデルが破綻する。
- 対象ブラウザ：Safari / Chrome（iPad・PC）。
- データ保存は **localStorage のみ**（端末内）。バックエンド・クラウド同期なし。
- 想定画面：**iPad 横向き、1 画面・スクロールなし**。

---

## 3. 通常モード（入力して使う）

### トップバー（ネイビー帯）
- 左：タイトル「重症喘息 コントロール整理シート」＋小さな補足「面談用・タップで選択」。
- 右：
  - 患者ラベル入力（プレースホルダ「患者イニシャル / 年齢」）
  - 記入カウンター「記入 n / 総数」
  - 「リセット」ボタン
  - 「編集」ボタン（モード切替）

### ボード（4列カード）
- 4 列のカードを横並び。各列＝**列見出し** ＋ 複数の**設問**。
- 設問＝**ラベル** ＋ **選択肢ボタンの行**。
- 選択肢の挙動：
  - タップで選択（**オレンジ塗り**）。
  - **単一選択**（同じ設問内で選べるのは1つ）。
  - 選択済みを再タップで**解除**。
- カウンター：「記入済みの設問数 / 全設問数」を常に反映（設問の増減に追従）。
- 「リセット」：全選択解除＋患者ラベルを空に。

---

## 4. 編集モード

トップバー「編集」でON/OFF（ラベルは「編集を終了」に変化、ボタンはアクティブ表示）。ONの間は編集用UIを表示。

- **編集ヒントバー**を表示：
  `編集モード：↑↓ で移動・✕ で削除・文字をタップで書き換え・＋ で追加。仕上がったら〔保存（書き出し）〕でファイルにして共有できます。`
- **設問ごと**：`↑` `↓`（列内で並び替え）、`✕`（その設問を削除）。
- **インライン編集**：ラベル・各選択肢・列見出しは**タップで直接書き換え**（contenteditable）。Enter で確定（改行は入れずにフォーカスを外す）。
- **追加**：各列の下に「＋ 項目を追加」、ボード右端に「＋ 列を追加」。
- **列削除**：列見出し横に「✕」。**確認ダイアログ**を出す（中の設問も消える旨）。
- **初期化**：「初期化」ボタン。**確認のうえ**、保存内容を消して初期セット（seed）に戻す。
- **保存（書き出し）**：現在の構成を反映した新しいHTMLをダウンロード（§6）。
- 編集モードに入るとき、**患者の選択状態はクリア**する（編集は構造の作業なので混同を防ぐ）。

> タッチ前提のため、並び替えは**ドラッグではなく ↑↓ ボタン**で行う（iPadで確実に動かすため）。

---

## 5. 保存・読み込み（localStorage）

- キー：`asthmaSheet_v1`
- **保存対象**：ボードの**構造**（列・設問・選択肢のテキスト）。**患者の選択状態は保存しない**。
- **保存タイミング**：構造変更時（移動・削除・追加）、テキスト編集の確定時（blur）、編集モード終了時。
- **読み込みの優先順位**：
  1. localStorage に保存があれば、それを復元。
  2. なければ、静的HTMLに**焼き込み済みの構成**（書き出しファイルの場合）があればそれを使う。
  3. どちらもなければ、**初期 seed** を描画。
- localStorage が使えない環境（サンドボックス等）では **try/catch で握りつぶし**、絶対にクラッシュさせない（保存されないだけにする）。

---

## 6. 書き出し（共有用HTML）

「保存（書き出し）」押下時：

1. 現在のドキュメントを複製し、以下を除去：編集用要素の表示状態（編集クラス）、`contenteditable` 属性、選択状態、患者入力値。
2. `<!doctype html>` を付けて**1ファイルのHTML文字列**を生成。
3. Blob でダウンロード（ファイル名例：`重症喘息シート_共有用.html`）。
4. 書き出したファイルは、開くと**焼き込み済みの構成で起動**する（受け取った人の localStorage が空でも、§5-2 で構成が表示される）。受け取った人もさらに編集・再書き出しできる。
5. ダウンロードできない環境では、その旨を**アラートで案内**（「ファイルをダウンロードしてブラウザで開いてから試す」旨）。

## 8. デザインシステム

### カラートークン（CSS変数）
```css
:root{
  --navy:#1B2A4A;  --navy-soft:#2C3E5E;
  --orange:#F37021; --orange-press:#E0641A;
  --bg:#EDF1F6;    --card:#FFFFFF;  --line:#D6DCE4;
  --ink:#1B2A4A;   --muted:#5B6675;
}
```
（GSKブランド：ネイビー #1B2A4A／オレンジ #F37021）

### タイポグラフィ
```
font-family: "Hiragino Sans","Hiragino Kaku Gothic ProN","Noto Sans JP",
             "Meiryo",-apple-system,BlinkMacSystemFont,"Segoe UI",sans-serif;
```
数値系（スコア・閾値）は `font-variant-numeric: tabular-nums;`。

### レイアウト要点
- 全体：縦フレックス。トップバー（高さ約 58px・固定）＋ボード（残り全部）。通常時は `height:100vh`。
- ボード：`display:flex; gap:14px; padding:16px;` の4列。各列 `flex:1`。
- 列カード：白背景、`border:1px var(--line)`、**上辺 3px**（通常ネイビー／アクセント列はオレンジ）、`border-radius:13px`、`padding:14px 13px`。
- 列見出し：下罫線あり、太字、ネイビー（アクセント列はオレンジ）。
- 設問：ラベル（12.5px・`--muted`・太め）＋選択肢行 `display:flex; gap:5px;`。設問は列内で縦に均等配置（`justify-content:center`）。
- 選択肢ボタン：`flex:1; min-height:40px; font-size:12px; border:1.5px var(--line); border-radius:8px;` 白地・ネイビー文字。テキストは折り返し可（行は高さを揃える）。
- 選択状態：背景 `--orange`／文字白／太字。ホバー：薄いグレー。フォーカス：オレンジのアウトライン。
- タッチ最適化：`-webkit-tap-highlight-color:transparent; touch-action:manipulation;`。

### 編集モードの見た目
- 編集中、ラベル・選択肢・列見出しに**破線アウトライン**を付け、編集できると分かるようにする。
- `↑ ↓ ✕`・列削除は 24px 角の小ボタン。削除系（✕）はホバーで赤系。
- 「＋ 項目を追加」は破線の横長ボタン、「＋ 列を追加」は破線の縦長タイル（列と同じ高さ）。
- 編集中はコンテンツが縦に伸びるため、`body` を `height:auto; min-height:100vh;` にしてスクロール可に。

### レスポンシブ・フォールバック
- `@media (max-width:1000px)`：4列を折り返し（各列 `flex:1 1 45%`）、`body` 高さ自動。
- `@media (max-width:560px)`：1列（`flex:1 1 100%`）。
- `@media (prefers-reduced-motion: reduce)`：トランジション無効。

## 9. アクセシビリティ／UX

- 選択肢は `role="button"` ＋ `tabindex="0"`、フォーカスリングを表示。
- 編集中の入力確定は Enter（フォーカスを外す）。
- iPad 横で**4列が 1 画面に収まり、スクロールが出ない**こと（通常モード）。
- タップ領域は十分（最小高さ 40px 目安）。

## 11. スコープ外

- 中央マスター型（全員で 1 枚を共同編集／変更が全員に反映）。これはホスティング＋共有保存（サーバー/DB）が必要で、本仕様の対象外。
- クラウド同期、ユーザー認証、印刷/PDF 最適化。

## 13. 実装の進め方（推奨順）

1. 通常モード：seed からボード描画 → 選択トグル → カウンター。
2. localStorage：保存・読み込み（優先順位 §5）。
3. 編集モード：移動・削除・テキスト編集・追加・列操作・初期化。
4. 書き出し：DOM複製→クリーニング→Blobダウンロード。

各段階を**単一HTMLファイルのまま**通すこと。既存 `asthma-control-sheet.html` を参照実装として渡せば、デザインと挙動を直接確認できる。
